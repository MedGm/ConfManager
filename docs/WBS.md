# Work Breakdown Structure (WBS)

## 1. Project Management
- **1.1 Initialization**
    - 1.1.1 Defines Scope & Objectives
    - 1.1.2 Team Setup (OBS)
    - 1.1.3 Tools Setup (GitHub, Jira/Trello, VSC)
- **1.2 Planning**
    - 1.2.1 WBS Creation
    - 1.2.2 Gantt Chart
    - 1.2.3 Budget Estimation
    - 1.2.4 Risk Analysis
- **1.3 Monitoring**
    - 1.3.1 Weekly Reviews
    - 1.3.2 Sprint Planning

## 2. Analysis & Design
- **2.1 Requirements**
    - 2.1.1 Functional Requirements (User Stories)
    - 2.1.2 Non-Functional Requirements
- **2.2 Modeling**
    - 2.2.1 Use Case Diagrams
    - 2.2.2 Class Diagrams / DB Schema
- **2.3 UI/UX Design**
    - 2.3.1 Wireframes (Figma)
    - 2.3.2 Prototype

## 3. Development
- **3.1 Environment Setup**
    - 3.1.1 Next.js Init
    - 3.1.2 Database Setup (Prisma)
- **3.2 Backend / API**
    - 3.2.1 Authentication (NextAuth)
    - 3.2.2 User API
    - 3.2.3 Event API
    - 3.2.4 Registration API
- **3.3 Frontend**
    - 3.3.1 Landing Page
    - 3.3.2 Auth Pages (Login/Register)
    - 3.3.3 Dashboard (Organizer)
    - 3.3.4 Dashboard (Participant)
    - 3.3.5 Event Details & Planning

## 4. Testing & Quality
- **4.1 Unit Testing**
- **4.2 Integration Testing**
- **4.3 User Acceptance Testing (UAT)**

## 5. Closure
- **5.1 Documentation**
    - 5.1.1 Technical Dossier
    - 5.1.2 User Manual
- **5.2 Final Presentation**
    - 5.2.1 Slide Deck
    - 5.2.2 Live Demo
